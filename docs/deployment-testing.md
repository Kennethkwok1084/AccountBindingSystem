# 部署、运维与测试

本文档整理项目上线前后最需要执行的部署、安全、恢复和测试要点，便于实施和交付。完整背景和历史方案见 [完整深度研究报告](./deep-research-report.md)。

## 1. 部署目标

- 部署环境为 Linux 内网。
- 采用 Docker Compose 管理服务。
- 使用同域 Nginx 承载前端并代理后端 API。
- 后端和调度器共享同一代码镜像，分离运行命令。
- 正式环境数据库统一为 PostgreSQL。

## 2. 服务清单

| 服务 | 作用 | 关键说明 |
|---|---|---|
| `postgres` | 数据存储 | PostgreSQL 16 |
| `backend` | Flask API | 生产模式关闭 Debug，启动时执行 migration |
| `scheduler` | APScheduler 任务进程 | 独立运行，禁止和 backend 重复起任务 |
| `nginx` | 反向代理与静态资源 | 同域部署，禁用目录浏览 |

## 3. 环境变量建议

| 变量 | 用途 |
|---|---|
| `ABS_DB_NAME` | PostgreSQL 数据库名 |
| `ABS_DB_USER` | PostgreSQL 用户 |
| `ABS_DB_PASSWORD` | PostgreSQL 密码 |
| `ABS_SECRET_KEY` | Flask 会话密钥 |
| `ABS_ADMIN_USERNAME` | 初始管理员用户名 |
| `ABS_ADMIN_PASSWORD` | 初始管理员密码 |
| `ABS_SESSION_COOKIE_SECURE` | Cookie 安全标志 |
| `TZ` | 时区 |

## 4. 安全基线

### 4.1 认证与会话

- 单管理员模型即可，不扩展多角色。
- 优先使用同域 `HttpOnly` Cookie。
- 所有写接口校验 `X-CSRF-Token`。
- 执行类接口校验 `X-Idempotency-Key`。

### 4.2 密码与文件

- 管理员密码优先使用 Argon2id 哈希存储。
- `scrypt` 仅保留为显式回退路径。
- 上传仅允许 `.xlsx`。
- 限制上传大小为 `10MB`。
- 服务器端重命名上传文件，下载接口必须鉴权。

### 4.3 应用与网关

- Flask 生产环境关闭 Debug。
- 统一返回 JSON 错误，不回显堆栈。
- Nginx 禁止目录浏览，并补充基础安全头。
- 下载行为写审计日志。

## 5. 备份与恢复

### 5.1 备份策略

| 类别 | 频率 | 方式 |
|---|---|---|
| 数据库全量备份 | 每天凌晨 | `pg_dump` |
| 上传文件备份 | 每天 | `rsync` 或卷快照 |
| 导出文件备份 | 每天 | `rsync` 或卷快照 |
| 配置文件备份 | 变更后 | Git 或只读备份 |

### 5.2 恢复 SOP

1. 停止 `backend` 和 `scheduler`。
2. 恢复最近可用 PostgreSQL 备份。
3. 恢复上传目录和导出目录。
4. 启动 `postgres` 并检查关键表计数。
5. 启动 `backend` 和 `scheduler`。
6. 验证当前绑定数量、最近台账、导出可下载、调度状态正常。

恢复演练记录请配合 [backup-restore-drill.md](./backup-restore-drill.md) 一起使用。

## 6. 测试策略

### 6.1 单元测试重点

| 场景 | 核心断言 |
|---|---|
| 包月顺延 | 到期日读取配置并增加对应天数 |
| 包年顺延 | 到期日读取配置并增加对应天数 |
| 导出文件名 | 满足 `移动YYMMDDHHmmss.xlsx`，冲突时自动补序号 |
| 导出空列 | 联通/电信列存在且为空 |
| 模板校验 | 列名、类型、重复行、值域校验准确 |

### 6.2 集成测试重点

| 场景 | 核心断言 |
|---|---|
| 账号池导入 | 建立批次和账号记录 |
| 导入任务详情 | 返回任务元数据与逐行错误 |
| 收费清单预览 | 正确分类 `allocate / renew / rebind / fail` |
| 收费清单执行 | 25 人场景下只成功前 20 人 |
| 完整名单导入 | 到期释放且冲突只预警 |
| 批次预警 | 到期批次能生成待办 |
| 批次换绑 | 执行后新号绑定、旧号处理、导出生成 |

### 6.3 端到端测试重点

| 场景 | 流程 |
|---|---|
| 新办学生全链路 | 登录 -> 导入账号池 -> 导入收费清单 -> 预览 -> 执行 -> 下载导出 |
| 异常账号手动换绑 | 登录 -> 查学号 -> 手动换绑 -> 单行导出 |
| 批次到期治理 | 生成预警 -> 进入待办 -> 预览换绑 -> 执行 |

## 9. 主要风险

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| 完整名单列名不稳定 | 导入失败 | 做模板校验与工作表配置 |
| 收费时间水位边界问题 | 可能漏单或重单 | 第一版保留现规则并记录升级项 |
| 长期号被过早消耗 | 长期库存被透支 | 明确批次优先级 |
| 批量执行重复提交 | 造成重复操作 | 强制幂等键 |
| 调度重复执行 | 造成重复释放或重复预警 | scheduler 独立部署 |
